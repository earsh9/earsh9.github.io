3.树状数组（Binary Index Tree, BIT）：折中/均衡的完成这两件事

最简单的树状数组支持这两种操作，时间复杂度均为 logn ：

和前缀和数组一样的想法，区间 [a,b] 的和 等于 [1,b] - [1,a)。问题转化为求前 n 项和

  • 用一个数组 C 维护若干个小区间：

    • 单点修改时，只更新包含这一元素的区间；

    • 求前n项和时，通过将区间进行组合，得到从1到n的区间，然后对所有用到的区间求和。

    • 设原数组是 A，如果 Ci 维护的区间是 [Ai,Ai]，此结构就相当于普通数组（还浪费了一倍内存）；如果 Ci 维护的区间就是 [1,Ai] ，此结构就相当于前缀和。

  • BIT 直译为 “二进制下标树”，说明与二进制息息相关。对于树状数组 C 中的任意下标 x：

    • 其能掌管的原数组 A 的区间和为：[x - lowbit(x) + 1, x]。这里取区间双闭，若想左开右闭，去掉 +1 即可。

    • lowbit()：x & (-x)；（-x 为 x 的补码）；若 x 最低一位为1的位数是 i，返回 2 ^ i

  • 例如：C[1] = A[1,1]、C[2] = A[1,2]、C[3] = A[3,3]、C[4] = A[1,4]、C[5] = A[5,5]、C[6] = A[5,6]、C[7] = A[7,7]、C[8] = A[1,8]

    • 可以看出奇数下标只管自己这一位的和，因为奇数的 lowbit 为 1，-1+1 直接抵消

    • 而偶数下标，像 2、4、8 这种只有一位是 1 的偶数，作差后直接掌管从头到它为止所有数的加和

  • 画成图就形如一颗二叉树，故此而得名。

4.线段树（segment tree）

  • 定义

    • 用来存放给定区间内对应信息的一种数据结构

    • 与树状数组相同，单点修改(update)和区间求和(range query)的时间复杂度都为 o(logn)

    • 不同的是还能进行区间最大值，区间最小值（Range Minimum/Maximum Query problem）或者区间异或值的查询

  • 实现原理

    • 数组：（与数组实现堆(heap)类似）用一个完全二叉树来存储对应于其每一个区间（segment）的数据

      • 索引从 1 开始，对于任意节点 i，其左子节点为 2i，右子节点为 2i+1，父节点为 i/2

    • 链表：

      • 动态开点，自顶向下初始化建树

      • 每一个结点中保存着相对应于这一个区间的信息：根节点保存[0,n)的和，叶子节点只存储单个元素信息，中间节点存储一段区间的信息（可以是区间和、区间最大值等…）

  • 实现步骤

    • 初始化建树

      • 自顶向下构建，从根节点（保存数组 [0,n) 或者 [1,n] 的信息）二分向下按层构建

      • 对于链表实现形式，只用创建根节点即可。随用随创建（lazyCreate）

    • 查询 query 和 修改 update 操作：

      • 自顶向下（不断 dfs 的过程）pushDown（将父节点保存的add懒标记信息传给孩子节点）

      • 递归到叶子节点时，再 pushUp

        • 按照不同的需求，对应两个子结点的数据相应的来构建存储于它们母结点中的值（取sum、max、min等…）

  • 单点修改(update)

    • 当输入数组中位于i位置的元素被更新时，我们只需从这一元素对应的叶子结点开始，沿二叉树的路径向上更新至根结点即可。（一个 dfs）

  • 区间查询

    • 若当前结点所代表的区间完全位于给定需要被查询的区间之外，则不考虑

    • 若完全位于给定需要被查询的区间之内，返回节点的值参与求和

    • 若部分位于需要被查询的区间之内，考虑区间内的（注意总有可能找到完全位于区间内的结点，因为叶子结点的区间长度为1，因此我们总能组合出合适的区间）

53. 最大子数组和

给你一个整数数组 nums ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

子数组 是数组中的一个连续部分。

示例 1：

输入：nums = [-2,1,-3,4,-1,2,1,-5,4]

输出：6

求解最长公共上升子序列的 pushUp 操作

  1. 定义一个操作 get(a, l, r) 表示查询 a 序列 [l, r] 区间内的最大子段和。

    • 本题就是 get(nums, 0, nums.length - 1)

  2. 如何分治？对于一个区间 [l, r]，取 m = (l + r) / 2

    • 对区间 [l, m] 和 [m + 1, r] 分治求解。当递归逐层深入直到区间长度缩小为 1 的时候，递归「开始回升」。

    • 考虑如何通过 [l, m] 区间的信息和 [m + 1, r] 区间的信息合并成区间 [l, r] 的信息。

  3. 对于一个区间 [l, r]，维护四个量：

    • lSum 表示 [l, r] 内以 l 为左端点的最大子段和

    • rSum 表示 [l, r] 内以 r 为右端点的最大子段和

    • mSum 表示 [l, r] 内的最大子段和

    • iSum 表示 [l, r] 的区间和

  4. 如何从 [l, m] （左子区间）和 [m + 1, r] （右子区间）的信息 合并到 [l, r]？

    • iSum 最容易，直接相加

    • lSum 考虑 max（左子区间的 lSum 、左子区间和 iSum + 右子区间 lSum）。rSum 同理。

    • mSum：考虑是否会跨越区间中间值 m ：(前两个不会，最后一个会)

    • max( 左子区间的 mSum、右子区间的 mSum、左子区间的 rSum + 右子区间的 lSum)
