取余基础

  • (a + b) % mod = ((a % mod) + (b % mod)) % mod
  • (a - b) % mod = ((a % mod) - (b % mod) + mod) % mod; //预防作差结果为负
  • (a * b) % mod = ((a % mod) * (b % mod)) % mod;

欧几里得算法：求最大公约数（gcd）

设 q 为 a、b 的最大公约数，记作 q = (a, b)；则 q = (a, b) = (b, a % b)。

证明：由定理有

a = k1 * q

b = k2 * q

要证第二个等号相等，即证 a % b 的最大公约数也是 q。

即设 a % b = r，求 r 是多少？

a = k * b + r

r = k1 * q - k * k2 * q = (k1 - k2 * k) * q。即证

    • 给出一个 a * x + b * y = c，可以求出 x 和 y

逆元

在实数域（R域）中，若 x * y = 1，则 x 与 y 互为逆元。

找逆元的方法：

  1. 扩展欧几里得
  2. 费马舍定理
  3. 欧拉定理

在取模的运算中，对任意 x % p 使得值在 [0,p) 范围内；p 为质数的情况下，1 ~ p - 1 的数均可找到逆元。

(2 * x) % p == 1，p = 7，求 x = ?

x = 4；(2 * 4) % 7 == 1。

2 与 4 互为逆元；1 的逆元为本身；3 与 5 互为逆元；6 的逆元为本身。

2156. 查找给定哈希值的子串

给定整数 p 和 m ，一个长度为 k 且下标从 0 开始的字符串 s 的哈希值按照如下函数计算：

hash(s, p, m) = (val(s[0]) * p0 + val(s[1]) * p1 + … + val(s[k-1]) * pk-1) mod m.

其中 val(s[i]) 表示 s[i] 在字母表中的下标，从 val(‘a’) = 1 到 val(‘z’) = 26 。

给你一个字符串 s 和整数 power，modulo，k 和 hashValue 。

请你返回 s 中 第一个 长度为 k 的 子串 sub ，满足 hash(sub, power, modulo) == hashValue 。

测试数据保证一定 存在 至少一个这样的子串。

子串 定义为一个字符串中连续非空字符组成的序列。
 
示例 1：

输入：s = “leetcode”, power = 7, modulo = 20, k = 2, hashValue = 0

输出：”ee”

解释：”ee” 的哈希值为 hash(“ee”, 7, 20) = (5 * 1 + 5 * 7) mod 20 = 40 mod 20 = 0 。

“ee” 是长度为 2 的第一个哈希值为 0 的子串，所以我们返回 “ee” 。

思路：

  1. 暴力选定窗口再计算哈希值会超限，考虑复用之前的计算的结果。

  2. 正向滑动窗口：

    • 第一个窗口：(x0*p^0 + x1*p^1 + ... + xk-1*p^k-1) % mod = value，若value != hashvalue

    • 第二个窗口：减去第一项再添加一项 xk*p^k；且现存的每一项都要除以 p，因为多了一次幂

  3. 问题在于 value 是取模后的结果，即 (a*p) % mod，需要知道 p 的逆元 x 才能保证前后系数不变：

    • (a * p * x) % mod == a
    • 例如：p = 2，mod = 7，需知道 2 的逆元为 4，才能保证在取模的情况下除以 p 并不改变系数的大小：
    • a * 2 * 4 % 7 = a % 7 * 8 % 7 = a

  4. 而若 mod 非质数，逆元不一定存在。如：2 * x % 20 = 1，求 x = ？

  5. 逆向滑动窗口采用乘以 p 的方式，取模运算中一样可以。

计算自然数 num 的数根

每个自然数都有一个数根。对于给定的自然数，反复将各个位上的数字相加，直到结果为一位数，则该一位数即为原自然数的数根

258. 各位相加

给定一个非负整数 num，反复将各个位上的数字相加，直到结果为一位数。返回这个结果。

除了O(n) 的模拟外，如何在 O(1) 的时间复杂度下实现？

  • 对于自然数 abc = 100 * a + 10 * b + c

  • 一次各位相加后为: a + b + c

  • 中间差值为 99 * a + 9 * b

  • 相当于 num 对 9 取模

  • 若取模结果为0，例如 99 % 9 == 0，9 + 9 -> 18，1 + 8 -> 9，直接return 9

  • 否则return 对 9 取模的结果，例如 83 % 9 = 2，8 + 3 -> 11 -> 2

当 num = 0 时，num − 1 = −1 < 0，负数对 9 取余或取模的结果的正负在不同语言中有所不同。

对于取余的语言，结果的正负和左操作数相同，则 num − 1 对 9 取余的结果为 −1，加 1 后得到结果 0，可以得到正确的结果；**(Java 为此)**

对于取模的语言，结果的正负和右操作数相同，则 num − 1 对 9 取模的结果为 8，加 1 后得到结果 9，无法得到正确的结果，此时需要对 num = 0 的情况专门做处理。
