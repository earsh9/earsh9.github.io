相交和环形链表

面试题 02.07. 链表相交

给定两个（单向）链表，判定它们是否相交并返回交点。请注意相交的定义基于节点的引用，而不是基于节点的值。换句话说，如果一个链表的第k个节点与另一个链表的第j个节点是同一节点（引用完全相同），则这两个链表相交。

  1. 对两个链表各循环一次，得到各自的长度
  2. 多的先走长度之差步，再一起走，直到指向同一个节点

142题 环形链表II

题意： 给定一个链表，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

为了表示给定链表中的环，使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。

最直观的思路，哈希表

Set<ListNode> visited = new HashSet<>()
依次记录每个节点，遇到重复的就返回

想空间复杂度o(1):

  1. 先判断是否有环形：快慢指针

    1. 快的每次比慢的多走一步，若有环形则在环内快的一定会一步一步追上慢的，即相遇。

  2. 若有环，找到这个环的入口：

    1. 设从头节点到环形入口节点 的节点数为x，环形入口节点到快慢指针相遇节点 的节点数为y，从相遇节点再到环形入口节点为z
    2. 在相遇时：slow指针走过的节点数为：x + y；fast指针走过的节点数为：x + y + n * (y + z)
    3. 二者又有二倍的关系，约分移项后：x = (n - 1) * (y + z) + z；

  3. 例如n = 1时，x = z，从头节点出发一个指针a，再从相遇节点出发一个指针b，这两个指针每次走一个节点，当两个指针相遇时即为环形入口节点。n > 1时即b要多走n - 1圈才遇见a。

  4. 为什么slow指针走过节点数为x + y而不是 x + y + 环的若干长度n呢？

    1. fast一定比slow先入环，当slow走到环形入口位置处时，fast一定在环内某个位置设其在环内已走过k个节点，k < n。
    2. 当fast下一次走到环形入口时，已走过k + n个节点，slow走过(k + n)/2。由于k < n，(k + n)/2 < n，即slow还没有走完一圈fast就已经走到了环形入口即相遇了。
    3. fast每次走两步slow每次走一步，fast是一步一步靠近slow，不存在会是跳过的情况。

或者说：

  • fast走过的节点数是slow走过的节点数的二倍，若slow和fast同时在环入口处出发，则slow走一圈fast走两圈就会再次相遇。
  • 而slow走到环入口时，fast一定在环中某一位置处，即fast多走了k个节点。slow还没走完一圈，二者就会再次相遇。
