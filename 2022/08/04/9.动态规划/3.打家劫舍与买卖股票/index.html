任意相邻的两间房屋，只能偷一间，问能偷到的最大金额是多少？

三道题不同之处在于房屋陈设布局不一样。

  1. 数组上连续元素二选一
  2. 成环之后连续元素二选一
  3. 房屋形似 “ 二叉树 ”，父与子节点不能同时偷

198题（确定状态转移方程

dp[j]：前 j 间房屋所能偷到的最大金额。

  1. 不偷，就延续 dp[j - 1] 的偷盗金额！
  2. 偷，dp[j - 2] 的金额 加上 当前房屋的金额

dp[j] = Math.max(dp[j - 1], dp[j - 2] + nums[j]);

  • 当前状态与 前一个 或者 前两个 状态有关。从前往后遍历
  • dp[0]与dp[1]需初始化 - > 只有一间房屋 和 两间房屋 可偷时应该怎么偷？

213题（拆段计算

由于房屋围成一圈，意味着第 1 间 和 第 n 间 不能 同时偷，问题转化为：

  • 第 1 ~ n - 1 间房屋，间隔偷盗
  • 第 2 ~ n 间房屋，间隔偷盗
  • 注意初始化的状态不一样！

写法 1 注重拆段，写法 2 注重 初始化

337题（dfs + 动规

  • 首先需确认 “遍历方式”：深度还是广度 ？
  • 父与子 之间的直接连接 两边只能选一个
  • 后序遍历 可以自由决定 抢父节点 还是考虑 抢孩子节点，并将返回的结果相加

暴力搜索 + 记忆化

避免重复计算 孙子节点的结果

树形 dp 入门题

  • 对于暴力递归，是实时计算 偷 与 不偷 的结果
  • 而 动规 用数组记录每个节点的结果就不用实时计算

买卖股票的最佳时机

121题：只能买卖一次

给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

示例 1：

输入：[7,1,5,3,6,4]

输出：5

解释：在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。

示例 2：

输入：prices = [7,6,4,3,1]

输出：0

解释：在这种情况下, 没有交易完成, 所以最大利润为 0

使用贪心，选取差值最大值

  • 暴力找最大间隔区间，超时
  • 贪心：股票只买卖一次，找左边最小值和右边最大值

动规：“持有” 不代表是当日买入，有可能是之前就有了

int[][] dp = new int[n][2]

  • dp[i][0] : 第 i 天 持有 股票所得的最多现金，买入即为 -prices[i]

    • dp[i][0] = Math.max(dp[i - 1][0], -prices[i]);

  • dp[i][1] ： 第 i 天 不持有 股票所得的最多现金

    • dp[i][1] = Math.max(dp[i - 1][1], dp[i - 1][0] + prices[i]);

  • 初始化：dp[0][1] = 0, dp[0][0] = -prices[0]

  • 还可以采用滚动数组优化

122题：多次交易，可以同一天买入和卖出

含手续费、冷冻期

可以无限次交易，但 「不能同时参与多笔交易」。且：

  • 714题 买卖股票的最佳时机含手续费

    • 每次卖出需支付手续费 fee

  • 309题 最佳买卖股票时机含冷冻期

    • 卖出股票后，你无法在第二天买入股票 (即冷冻期为 1 天)。

规定最多可交易的笔数

依旧是 「不能同时参与多笔交易」

  • 123题：最多两笔

    • 定义五种状态：

      • 0: 没有操作, 1: 第一次买入, 2: 第一次卖出, 3: 第二次买入, 4: 第二次卖出

    • 边界初始化：

      • 第二次买入：依赖于第一次卖出的状态，相当于第0天第一次买入，第一次卖出，然后在买入一次（第二次买入），那么现在手头上没有现金，只要买入，现金就做相应的减少。

    • 在最终时，卖出一定比持有现金多；而两次卖出的状态现金最大一定是最后一次卖出。

  • 188题：最多 k 笔

  • 更加规律化的 k 笔交易

    • 初始化所有的交易次数是为确保 最后结果是最多 k 次买卖的最大利润

  • 简化 dp 数组的维度：

    • 持有 和 不持有 的状态由 buy 和 sell 数组分别表示

      • buy[i][j]：对于数组 prices[0..i] 中的价格，进行了 j 笔交易，且处于 持有 状态

      • sell[i][j]：处于 不持有 状态

    • 第 i 天的状态 只与 前一天 有关：使用滚动数组

      • 删除 n 那一维度即可（未贴出）
