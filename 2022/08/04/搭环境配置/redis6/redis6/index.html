问题记录

因为给安装 linux 系统的服务器设置了IP地址：192.168.1.21，所以每次远程登录就行：ssh oem@192.168.1.21，密码abc123。无需使用 vmware

问题记录

  1. Linux系统（Ubuntu） 中安装 redis6

    • 使用 make 编译完，想使用 make install 安装时， 报 error 详情为 Permission denied（权限否定）

      • 说明给的权限不够，前加 sudo 即可

    • cd 后啥都不加表示进入当前用户的家目录下。使用 pwd 可以看到完整目录：/home/oem（此为当前用户），说明home目录下还有很多其他文件夹

      • cd /usr/…：cd 后直接加一个 / 表示从最顶端的目录开始往下找（即home目录），通过 cd home、ls 后确实可以看到一个 usr (即user) 目录

    • 为防止出问题，可以给redis设置密码（这里是abc123）。见https://www.php.cn/redis/422222.html

    • 启动方式：进入 bin 目录、使用命令、查看进程、通过客户端连接

     因为前面设置了密码，还需要 auth abc123 才能使用

基本解析

  1. redis是 单线程 + 多路IO复用

    • 具有原子性，操作一旦开始就会运行到结束。不会被其他线程所打断，因为是单线程嘛
