git常见指令&配置

Ubuntu 中配置 Git

设置用户名和邮箱

git config –global user.name 用户名（github官网注册的用户名）

git config –global user.email 邮箱（gitub官网注册绑定的邮箱）

查看配置：git config –list

配置 SSH

SSH 可以配合大文件传输。

ssh-keygen -t rsa -C "xxx@gmail.com"

提示的地方一路 enter，注意-keygen 前没有空格。

查看密钥并去 Github 上配好

cat ~/.ssh/id_rsa.pub

RPC failed; curl 56 GnuTLS recv error (-54): Error in the pull function.

把 git clone 方式从 https 改为 ssh 解决

常见指令

$ ssh-keygen -t rsa -C “youremail@example.com“ //获取 SSH key，将其填在 gitee/github 以此来识别内容是谁push上去的

创建与修改

sudo apt install git //安装xxx软件

git config –global user.name / user.email //为当前git分布式管理系统设置一个名字

vim readme.txt //git 严格区分大小写，vim相当于linux中的一个记事本。即拿vim打开.txt文件

本地代码都保存在工作区。先 add 到缓存区，后 commit 到本地的库中，此时可以查看整个分支的增删改信息。最后 push 到远程库中，相关的 merge 操作还会有管理权限限制

commit -m 后添加本次修改的备注信息，建议按实际填写

在 add 前可以通过 git status 查看当前修改信息 或者 git diff xxx文件 来具体查看对文件的修改信息

版本回退

git log //显示从最近到最远的提交日志，想精简点后加 –pretty=oneline

git reflog //查看 commit_id 即版本号的，方便进行删除和撤销删除

git reset –hard commit_id //有 hard 就是删除，没有就是撤销删除

远程库

git remote -v //查看远程库的信息

git remote rm origin //删除与远程库的联系（远程库本身没有删除）

git remote add origin git@gitee.com:earsh/learngit.git //与远程库建立联系

分支管理

git switch master //切换到 master 分支

git switch -c dev //创建并切换到 dev 分支

git branch //查看当前所在的分支（会罗列出所有分支，带*号的是当前所在分支）

git merge dev //用于合并指定分支到当前分支。即将 dev 分支合并到 master 分支（合并后可以删除dev分支：git branch -d dev）
